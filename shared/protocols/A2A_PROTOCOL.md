# OPT 龙虾军团 A2A 协作协议 v3.0

## 1. 核心原则：物理隔离与两步触发

为了防止多个 Agent 在同一个上下文中互相干扰，OPT 军团采用**物理隔离的沙箱环境**和**两步触发协议**。

### 1.1 频道隔离
每个 Agent 绑定一个独立的 Slack 频道（或飞书群组）：
- `#hq`：老板与 CoS 的专属频道。
- `#cto`：技术讨论频道。
- `#build`：代码提交和部署频道。
- `#research`：市场调研和竞品分析频道。
- `#know`：知识沉淀和技能提炼频道。
- `#ops`：系统监控和安全审计频道。

### 1.2 两步触发 (Two-Step Trigger)
当一个 Agent 需要另一个 Agent 协助时，不能直接在后台调用，必须遵循以下步骤：

1. **可见锚点**：发起方在目标频道的可见消息中，使用 `@AgentName` 明确呼叫目标 Agent，并简述任务。
2. **触发执行**：发起方调用 `sessions_send` 工具，将完整任务内容发送给目标 Agent。

这种机制确保了**所有跨 Agent 的协作都在人类老板的视线范围内**，防止 AI 团队在后台“失控”。

## 2. 任务传递规范

### 2.1 任务结构 (QAPS)
所有跨 Agent 的任务传递，必须遵循 QAPS 结构：
- **Q (Question)**：任务的核心问题或目标。
- **A (Action)**：期望的具体行动或产出。
- **P (Context/Parameters)**：相关的背景信息、约束条件或参数。
- **S (Success Criteria)**：明确的验收标准。

### 2.2 状态反馈
接收方在完成任务后，必须在目标频道回复发起方，并附上：
- 任务执行结果的简要总结。
- 相关的产出文件或链接。
- 遇到的问题及解决方案（如有）。

## 3. 异常处理

### 3.1 任务超时
如果接收方在规定时间内未完成任务，发起方应：
1. 在目标频道发送提醒消息。
2. 如果仍无响应，向 CoS 或 CTO 汇报异常情况。

### 3.2 任务失败
如果接收方在执行任务时遇到无法解决的错误，应：
1. 立即停止执行，避免造成进一步破坏。
2. 在目标频道向发起方汇报详细的错误日志和已尝试的方案。
3. 等待发起方或人类老板的进一步指示。
