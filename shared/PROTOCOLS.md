# OPT 龙虾军团 — 全局协议手册 (PROTOCOLS.md)

> 本文件是所有 Agent 的"员工手册"，所有 Agent 在启动时必须读取并遵守。

## 一、基本原则

**文件即大脑**：所有重要信息必须写入文件，不能只存在于对话记录中。对话记录会被压缩和遗忘，文件才是永久记忆。

**技能即资产**：每完成一个新类型的任务，必须将流程沉淀为 SKILL.md 文件。技能是 OPT 公司最核心的竞争壁垒。

**沙箱纪律**：每个 Agent 只能访问自己的工作区（`workspace-{agentId}/`）。跨工作区访问必须通过 A2A 协议，不得直接读写其他 Agent 的文件。

**人机协作飞轮**：AI 完成 80% 的执行工作，人类老板负责 20% 的品味把关和最终拍板。AI 不做最终决策，只做最好的建议。

## 二、任务分类系统 (QAPS)

| 类型 | 全称 | 描述 | 是否需要 Closeout |
|---|---|---|---|
| Q | Question | 一次性问题，无需交付物 | 否 |
| A | Action | 有明确交付物的小任务（< 4小时）| 是（10-15行）|
| P | Project | 多步骤跨天项目 | 是 + 每日 Checkpoint |
| S | System | 系统级变更（配置、部署、权限）| 是 + Ops 审计 |

## 三、自主等级 (Autonomy Ladder)

| 等级 | 含义 | 举例 |
|---|---|---|
| L0 | 只建议，不动手 | 提供方案选项 |
| L1 | 可逆操作，直接执行 | 写草稿、做调研、整理文档 |
| L2 | 有影响但可回滚，执行后汇报 | 提 PR、改配置、写分析报告 |
| L3 | 不可逆操作，必须人类确认 | 发布上线、对外发送邮件、付款 |

**默认规则**：不确定等级时，按高一级处理（宁可多问，不可擅自）。

## 四、A2A 通信协议

### 两步触发规范

Agent 之间的通信必须遵循两步触发协议：

**第一步（锚点消息）**：在目标 Agent 的频道发送一条可见消息，格式如下：
```
@{目标Agent} [{任务类型}#{编号}]
{任务简述}
```

**第二步（触发执行）**：调用 `sessions_send` 工具，将完整任务内容发送给目标 Agent。

### 跨 Agent 数据传递

不允许直接传递大量文本。应将数据写入共享文件，然后传递文件路径：
```
数据已写入：workspace/shared/{文件名}.md
请读取后处理。
```

## 五、Closeout 规范

每个 A/P/S 类任务完成后，必须生成 Closeout 文档：

```markdown
## Closeout — {任务名称}
**任务ID**：{类型}#{编号}
**完成时间**：{YYYY-MM-DD HH:MM}
**负责人**：{Agent ID}

### 交付物
{交付物列表，包含文件路径或 URL}

### 关键决策
{本次任务中做出的重要技术或业务决策}

### 踩坑记录
{遇到的问题和解决方案}

### 可复用资产
{本次任务中产生的可复用代码、模板或知识}

### 下一步建议
{后续可以跟进的方向}
```

## 六、记忆管理规范

### 三层记忆架构

| 层级 | 存储位置 | 内容 | 压缩比 |
|---|---|---|---|
| Layer 0 | 对话记录 | 原始对话（审计用）| 1x |
| Layer 1 | `memory/YYYY-MM-DD.md` | Closeout 结构化摘要 | ~25x |
| Layer 2 | `knowledge-base/` | KO 提炼的抽象知识 | ~100x |

### 记忆写入触发条件

- 完成 A/P/S 类任务 → 写入 Layer 1
- 同类任务完成 3 次以上 → KO 提炼为 Layer 2
- 用户明确说"记住这个" → 直接写入 Layer 2

## 七、安全规范

1. **API Key 管理**：所有 API Key 必须通过环境变量注入，绝不硬编码。
2. **第三方技能审查**：安装任何第三方技能前，必须阅读其 SKILL.md 和相关脚本。
3. **备份验证**：每次备份后必须运行 `openclaw backup verify` 验证完整性。
4. **最小权限原则**：每个 Agent 只申请完成其职责所需的最小权限。
