# OPT AutoDream 记忆巩固系统

## 技能描述
本技能基于 Claude Code 的四层记忆架构（CARROS），为 KO（知识管理员）提供一套标准的后台记忆整理机制。它模拟人类睡眠时的记忆巩固过程，在空闲时自动整理、萃取和索引记忆碎片，实现知识的长期沉淀。

## 适用场景
- 定期（如每天凌晨）整理团队的日常工作总结（Closeout 摘要）。
- 将碎片化的经验提炼为可复用的知识资产。
- 维护和更新 RAG 知识库。

## 执行流程

### 1. 碎片收集（Gather）
- 读取各 Agent 的 L1 记忆文件（`workspace-{agentId}/memory/YYYY-MM-DD.md`）。
- 收集当天的项目进展、遇到的问题、解决方案和代码片段。

### 2. 关联分析（Analyze）
- 找出这些碎片之间的关联，例如之前的配置问题和现在的报错的关系。
- 识别重复出现的模式或错误。

### 3. 知识萃取（Extract）
- 将碎片化的信息提炼成结构化的知识点。
- 提取设计原则、代码规范、操作手册等核心内容。
- 如果某个任务流程被成功执行 3 次以上，将其提炼为 `SKILL.md` 文件。

### 4. 记忆索引（Index）
- 将萃取的知识点向量化，存入 L2 知识库（`knowledge-base/` 目录）。
- 更新知识库的索引文件，方便后续检索。

## 记忆管理原则
- **观点而非事实**：记忆是观点而非事实，不保存可通过工具实时推导的信息（如代码模式、Git 历史）。
- **严格分离**：将记忆与计划/任务严格分离，避免混淆。
- **去重优化**：自动合并重复的知识点，保持知识库的精简和高效。
