# OPT TAOR 引擎 (Think-Act-Observe-Repeat)

## 技能描述
本技能基于 Claude Code 的核心运行机制（TAOR 引擎），为 Builder 和 Researcher 提供一套标准的复杂任务执行框架。它确保 Agent 在遇到错误时能够自动捕获输出、分析原因并重试，而不是直接向用户报错。

## 适用场景
- 执行复杂的系统命令或脚本。
- 编写和调试代码。
- 进行深度的网页抓取或数据分析。

## 执行流程

### 1. Think（思考）
- 分析当前状态和目标。
- 制定下一步的具体行动计划。
- 预判可能出现的错误和风险。

### 2. Act（行动）
- 调用底层原语（如 `shell`、`file`、`browser` 工具）。
- 执行具体的命令、代码或操作。

### 3. Observe（观察）
- 读取执行结果（标准输出、错误输出、文件内容、网页状态）。
- 评估是否报错，判断目标是否达成。

### 4. Repeat（重复）
- 如果目标未达成或出现错误，分析原因。
- 调整策略，修改代码或命令。
- 返回第 2 步（Act），继续循环，直到成功或达到最大重试次数（默认 3 次）。

## 错误处理原则
- **不轻易放弃**：遇到错误时，必须至少尝试 2 种不同的解决方案。
- **不掩盖问题**：如果达到最大重试次数仍未成功，必须向 CoS 或 CTO 汇报详细的错误日志和已尝试的方案。
- **Fail-closed**：在重试过程中，如果涉及危险操作（如删除文件、修改配置），必须暂停并请求确认。
