# 技能：OPT 每日工作报告

## 简介
本技能为 CoS（幕僚长）提供**每日工作总结生成**能力。它在每个工作日结束时，自动收集各 Agent 的工作成果，生成结构化的日报，并触发 KO 的记忆巩固流程（Auto Dream），确保重要信息沉淀到 L2 知识库。

## 适用角色
**CoS**（主要）、**KO**（协同）

## 触发条件
- 每个工作日结束时（推荐 18:00）自动触发。
- 当老板主动要求"今天做了什么"时手动触发。
- 在完成重要项目里程碑后手动触发。

## 执行步骤

### 步骤 1：收集工作成果（Collect）
从各 Agent 的工作区收集今日成果：
```bash
# 查看今日修改的文件
find ./workspace -newer $(date +%Y-%m-%d) -type f | head -20

# 查看今日的 Git 提交记录
git log --since="$(date +%Y-%m-%d) 00:00" --oneline
```

### 步骤 2：生成日报（Generate）
按照以下模板生成日报，保存到 `workspace/cos/daily-YYYY-MM-DD.md`：

```markdown
📊 OPT 龙虾军团日报 - YYYY-MM-DD

## 今日完成

### Builder 完成
- [任务描述] - [产出物链接或路径]

### Researcher 完成
- [调研主题] - [关键发现摘要]

### KO 完成
- [知识沉淀内容] - [知识库路径]

## 明日计划
- [任务描述] - [负责 Agent]

## 关键决策记录
| 决策内容 | 决策理由 | 决策时间 |
|---|---|---|
| [描述] | [理由] | [时间] |

## 待解决问题
- [问题描述] - [预计解决时间]

## 今日学习
[今天学到的新知识或新工具，将由 KO 沉淀到知识库]
```

### 步骤 3：触发记忆巩固（Auto Dream）
日报生成后，通知 KO 执行 Auto Dream 记忆巩固：
- 将今日的重要发现和决策从 L1 记忆提炼到 L2 知识库。
- 更新用户画像（如果今日观察到新的偏好变化）。
- 检查是否有新的技能需要沉淀。

### 步骤 4：发送报告（Deliver）
将日报通过配置的渠道发送给老板：
```bash
# 如果配置了 Slack，发送到 Slack
if [ -n "${SLACK_BOT_TOKEN}" ]; then
  curl -X POST -H "Authorization: Bearer ${SLACK_BOT_TOKEN}" \
    -H "Content-Type: application/json" \
    -d "{\"channel\": \"${SLACK_CHANNEL}\", \"text\": \"📊 OPT 龙虾军团日报已生成：workspace/cos/daily-$(date +%Y-%m-%d).md\"}" \
    https://slack.com/api/chat.postMessage
fi
```

## 日报质量标准

| 质量维度 | 标准 |
|---|---|
| 完整性 | 覆盖所有活跃 Agent 的工作成果 |
| 简洁性 | 每个任务描述不超过 50 字 |
| 可追溯性 | 每个成果都有对应的文件路径或链接 |
| 前瞻性 | 明日计划清晰，责任人明确 |

## 注意事项与避坑指南
- **不夸大成果**：日报中只记录真实完成的任务，不记录"正在进行中"的任务。
- **记录失败**：如果某个任务失败了，也要如实记录，并说明原因和下一步计划。
- **保持简洁**：老板不关心技术细节（如 Token 消耗），只关心业务进展和决策需求。
