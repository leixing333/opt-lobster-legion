# OPT 每日报告技能

## 技能描述
本技能为 CoS（幕僚长）提供一套标准的每日汇报模板，确保老板每天早上 8 点能够收到简洁、有价值的团队状态报告。

## 报告模板

```
📊 OPT 每日简报 — {YYYY-MM-DD}

🔍 灵感嗅探
[今日值得关注的行业动态或设计趋势，1-2 条]

⚙️ 生产感知
[当前进行中的项目状态，哪里卡住了]

📦 资产沉淀
[今天新增了哪些可复用的知识或技能]

🎯 今日重点
[今天最需要老板关注或决策的 1 件事]
```

## 执行流程

### 1. 数据收集
CoS 在每天早上 7:50 触发报告生成流程：
- 从各 Agent 的 L1 记忆文件（`workspace-{agentId}/memory/YYYY-MM-DD.md`）中收集昨日进展。
- 从 Researcher 的情报库中提取最新的行业动态。
- 汇总 Ops 的系统健康状态。

### 2. 报告生成
CoS 根据收集到的数据，填写报告模板，生成简洁、有价值的每日简报。

### 3. 报告发送
CoS 在每天早上 8:00 将报告发送到 `#hq` 频道，并 @老板。

## 注意事项
报告应保持简洁，每条内容不超过 2 行。老板不关心技术细节（如 Token 消耗、Python 脚本），只关心业务进展和决策需求。
