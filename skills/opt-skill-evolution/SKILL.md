# 技能：OPT 技能自进化引擎 (Skill Evolution Engine)

## 1. 技能简介
本技能实现了基于 Hermes Agent 架构的“技能自进化”机制。它允许 OPT 龙虾军团的 Agent（特别是 KO 知识管理员和 Builder）在完成复杂任务后，自主将解决方案沉淀为标准化的 `SKILL.md` 文件，并在后续使用中不断优化和迭代这些技能。

## 2. 触发条件
- 当 Agent 成功解决了一个具有复用价值的复杂问题时。
- 当 Agent 在使用现有技能时发现步骤缺失、错误或有更优解时。
- 当 CoS（幕僚长）明确指示需要将某项工作流固化为标准技能时。

## 3. 核心机制：正反馈循环
本技能遵循 Hermes Agent 的闭环学习哲学：
**创建 (Create) → 使用 (Use) → 发现改进点 (Identify) → 更新 (Update) → 进化 (Evolve)**

## 4. 执行步骤

### 步骤 1：技能识别与创建 (Create)
1. **评估复用性**：判断刚刚完成的任务是否可能在未来再次出现。如果是，则进入技能创建流程。
2. **提取核心逻辑**：从对话历史和执行轨迹中，提取出解决该任务的关键步骤、所需工具、前置条件和注意事项。
3. **格式化输出**：严格按照 `agentskills.io` 开放标准，生成 `SKILL.md` 文件。
4. **保存技能**：将生成的 `SKILL.md` 保存到 `skills/<skill-name>/` 目录下。

### 步骤 2：技能使用与验证 (Use & Verify)
1. **加载技能**：在遇到类似任务时，优先检索并加载相关的 `SKILL.md`。
2. **严格执行**：按照技能文档中定义的步骤执行任务。
3. **记录偏差**：在执行过程中，记录实际情况与技能文档描述不符的地方（例如：某个 API 接口已变更，某个步骤需要额外的参数）。

### 步骤 3：技能迭代与优化 (Update)
1. **触发更新**：如果步骤 2 中记录了偏差或发现了更优解，立即触发技能更新流程。
2. **修改文档**：使用文件编辑工具，直接修改对应的 `SKILL.md` 文件，修正错误、补充细节或优化流程。
3. **版本记录**：在技能文档的末尾（或专门的 CHANGELOG 区域）简要记录本次更新的内容和原因。

## 5. `SKILL.md` 标准模板 (agentskills.io 兼容)

```markdown
# 技能名称：[简明扼要的名称]

## 简介
[一句话描述该技能的作用和适用场景]

## 触发条件
- [条件 1]
- [条件 2]

## 前置准备
- [需要的环境变量、API Key 或特定文件]

## 执行步骤
1. **[步骤 1 名称]**：[详细操作说明]
2. **[步骤 2 名称]**：[详细操作说明]

## 注意事项与避坑指南
- [容易出错的地方]
- [安全限制或边界条件]

## 变更日志 (Changelog)
- [YYYY-MM-DD] 初始创建。
- [YYYY-MM-DD] 优化了步骤 X，修复了 Y 问题。
```

## 6. 安全与边界
- **人工审计**：所有自动创建或更新的技能，必须在 Ops（运维审计员）的每周审计报告中列出，供人类老板（User）Review。
- **破坏性操作限制**：技能文档中不得包含未经授权的破坏性命令（如 `rm -rf /`、删库等），必须遵循 Fail-closed 安全原则。
