# OPT 安全审计系统

## 技能描述
本技能基于 Claude Code 的 Fail-closed 安全原则，为 Ops（运维审计员）提供一套标准的系统健康监控和角色漂移防护机制。它确保所有操作都有完整的日志溯源，防止“Undercover Mode”（隐身模式）带来的审计断层风险。

## 适用场景
- 每周检查各 Agent 的对话摘要，对照其 `SOUL.md` 判断是否越界。
- 监控关键操作（如网站发布、系统配置更改），确保所有操作都有完整的日志溯源。
- 审计系统健康状态，发现潜在的安全风险。

## 执行流程

### 1. 角色漂移防护（Audit）
- 读取各 Agent 的 L1 记忆文件（`workspace-{agentId}/memory/YYYY-MM-DD.md`）。
- 对照其 `SOUL.md`，检查是否存在越界行为（如 Builder 擅自修改架构设计，或 CoS 直接写代码）。
- 记录违规操作，并向 CoS 或 CTO 汇报。

### 2. 操作监控（Monitor）
- 监控关键操作（如网站发布、系统配置更改），确保所有操作都有完整的日志溯源。
- 检查是否存在“Undercover Mode”（隐身模式）带来的审计断层风险。
- 记录异常操作，并向 CoS 或 CTO 汇报。

### 3. 系统健康检查（Check）
- 检查系统资源使用情况（如 CPU、内存、磁盘空间）。
- 检查系统日志，发现潜在的安全风险。
- 记录异常情况，并向 CoS 或 CTO 汇报。

## 安全原则
- **Fail-closed**：所有安全相关的默认值都是最保守的。工具默认不可并行、默认非只读、权限默认需要确认。
- **操作全链路审计**：危险操作强制弹窗，隐身模式（Undercover）行为监控。
- **工具即能力边界**：Agent 能做什么完全由工具集决定，没有任何后门。
