# 技能：OPT 辩证式用户建模 (Dialectical User Modeling)

## 1. 技能简介
本技能基于 Hermes Agent 的 Honcho 引擎理念，为 OPT 龙虾军团引入“辩证式用户建模”能力。它允许 Agent（特别是 CoS 幕僚长和 KO 知识管理员）在与人类老板的长期交互中，不仅记录静态事实，更能捕捉用户的偏好演变、动机变化和深层需求，构建一个动态进化、避免自相矛盾的用户画像。

## 2. 触发条件
- 当 CoS（幕僚长）在对话中观察到人类老板提出了新的偏好、习惯或业务方向时。
- 当人类老板的指令与现有的用户画像（正题）发生冲突或补充时。
- 在每个工作日结束（Closeout）或项目里程碑节点时。

## 3. 核心机制：正-反-合 (Thesis-Antithesis-Synthesis)
本技能借鉴黑格尔的辩证法逻辑，通过以下三个阶段更新用户画像：
1. **正题 (Thesis)**：当前对用户的理解（即现有的用户画像）。
2. **反题 (Antithesis)**：在最新对话或任务中观察到的、与当前理解矛盾或补充的新信息。
3. **合题 (Synthesis)**：利用 LLM 的推理能力，将正题和反题融合，生成一个更新的、更准确的自然语言画像。

## 4. 执行步骤

### 步骤 1：观察与提取 (Antithesis Extraction)
1. **持续监听**：在日常交互中，留意人类老板关于工具偏好、代码风格、沟通方式、业务重心的表述。
2. **提取反题**：将这些新观察到的事实提取出来。例如：“老板今天要求用 Rust 重写一个原本用 Python 写的模块，并强调了内存安全。”

### 步骤 2：检索当前画像 (Thesis Retrieval)
1. **读取画像**：从 `knowledge-base/profiles/USER_PROFILE.md` 中读取当前的用户画像（正题）。例如：“老板是一个经验丰富的 Python 开发者，偏好快速原型开发。”

### 步骤 3：辩证融合 (Synthesis Generation)
1. **LLM 推理**：将正题和反题同时输入给 LLM（通常由 KO 角色执行），要求其进行辩证融合，而不是简单的事实堆砌。
2. **生成合题**：LLM 生成新的自然语言画像。例如：“老板是一个有深厚 Python 背景的开发者，目前正在向系统级编程（Rust）转型，越来越关注代码的性能和内存安全性，但在快速验证想法时仍会使用 Python。”

### 步骤 4：更新画像库 (Profile Update)
1. **覆盖更新**：将生成的合题（新画像）覆盖写入 `knowledge-base/profiles/USER_PROFILE.md`。
2. **版本记录**：在画像文件的末尾记录本次更新的简要原因和时间戳。

## 5. 画像数据结构 (自然语言描述)
与传统的键值对（Key-Value）不同，辩证式画像采用纯自然语言描述，以便直接注入到 Agent 的系统提示（System Prompt）中。

**示例 `USER_PROFILE.md` 结构：**
```markdown
# 用户画像：[老板姓名/代号]

## 核心画像 (The Synthesis)
[一段连贯的自然语言描述，综合了用户的技术栈、偏好、动机和近期发展趋势。]

## 沟通偏好
- [例如：喜欢简短直接的回答，附带代码示例，讨厌长篇大论的理论解释。]

## 近期关注点
- [例如：正在探索 WebAssembly 和边缘计算。]

## 演变日志 (Evolution Log)
- [YYYY-MM-DD] 观察到用户开始关注 Rust 内存安全，更新了核心画像。
- [YYYY-MM-DD] 初始画像建立（Python 开发者）。
```

## 6. 上下文注入 (Context Injection)
- **首轮对话**：CoS 在开启新会话时，应主动读取 `USER_PROFILE.md` 的核心内容，并将其作为上下文参考，以便从一开始就“认识”这个不断进化的老板。
- **避免缓存失效**：在长对话中，如果画像发生微调，尽量将更新后的画像附加到用户消息中，而不是修改系统提示，以最大化利用 LLM 的 Prompt Caching 机制（参考 Hermes 的优化策略）。
