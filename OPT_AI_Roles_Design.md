# OPT 一人设计公司 AI 角色体系与系统架构设计

## 1. 架构概述

基于 OpenClaw 和 OpenCrew 的多 Agent 协同理念，我们将 OPT 一人设计公司构建为一个“数字写字楼”。该架构采用三层设计，包含 6 个核心 AI 角色，通过标准化的文件交换和异步队列进行协同。

### 1.1 三层架构设计

1.  **意图对齐层（指挥中心）**：负责理解用户（人类老板）的深层意图，拆解目标，并验收最终结果。
2.  **执行层（业务部门）**：负责具体的业务落地，包括技术架构、代码编写、市场调研和设计产出。
3.  **系统维护层（后勤保障）**：负责知识沉淀、系统审计和防止 Agent 行为漂移。

## 2. 核心 AI 角色定义 (The 6-Agent Swarm)

### 2.1 CoS (Chief of Staff - 幕僚长)
*   **定位**：意图对齐层，用户的唯一直接对话窗口。
*   **职责**：接收模糊指令，转化为结构化需求；向执行层派发任务；汇总各部门进度并向用户汇报。
*   **模型建议**：Claude 3.5 Sonnet 或 GPT-4o（高智商，强逻辑）。
*   **频道绑定**：`#hq` (Headquarters)

### 2.2 CTO (Chief Technology Officer - 技术合伙人)
*   **定位**：执行层，技术大脑。
*   **职责**：接收 CoS 的需求，进行技术选型和架构设计；将大任务拆解为具体的开发工单（Tickets）；Code Review。
*   **模型建议**：Claude 3.5 Sonnet。
*   **频道绑定**：`#cto`

### 2.3 Builder (执行者/全栈工程师)
*   **定位**：执行层，无情的打字机。
*   **职责**：接收 CTO 的工单，编写代码，执行 CLI 命令，部署网站；严格遵守沙箱纪律，不出错是唯一 KPI。
*   **模型建议**：Qwen-Max 或 Kimi（性价比高，速度快）。
*   **频道绑定**：`#build`

### 2.4 Researcher (研究员/市场分析师)
*   **定位**：执行层，情报雷达。
*   **职责**：突破反爬机制抓取竞品数据；分析行业趋势；为设计和开发提供数据支撑。
*   **模型建议**：Kimi（长文本处理能力强）。
*   **频道绑定**：`#research`

### 2.5 KO (Knowledge Officer - 知识管理员)
*   **定位**：系统维护层，企业海马体。
*   **职责**：从日常对话和任务产出中提炼可复用的知识（原则、模式、踩坑记录）；更新全局知识库（RAG）。
*   **模型建议**：GPT-4o-mini。
*   **频道绑定**：`#know`

### 2.6 Ops (Operations - 运维审计员)
*   **定位**：系统维护层，纪律委员。
*   **职责**：审计系统变更；监控 Token 消耗；防止 Agent 角色认知精神分裂（角色漂移）；定期执行安全备份。
*   **模型建议**：Qwen-Max。
*   **频道绑定**：`#ops`

## 3. 协同机制与通信协议 (A2A Protocol)

为了避免多个 Agent 在同一个上下文中互相干扰（上下文爆栈），OPT 系统采用物理隔离的沙箱环境。

### 3.1 频道隔离与两步触发
*   **频道隔离**：每个 Agent 绑定独立的 Slack/飞书 频道（如 `#hq`, `#cto`）。
*   **两步触发 (A2A)**：Agent 之间不能直接在后台无限对话。当 CTO 需要 Builder 干活时：
    1.  CTO 在 `#build` 频道发送一条可见消息（锚点），例如：“@Builder，请执行工单 #123”。
    2.  CTO 调用 `sessions_send` 工具真正触发 Builder。
    3.  Builder 在该消息的 Thread（任务隔离）中回复执行结果。

### 3.2 任务分类 (QAPS)
*   **Q (Question)**：一次性问题，无需总结。
*   **A (Action)**：有交付物的小任务，完成后需生成 Closeout（10-15行结构化总结）。
*   **P (Project)**：多步骤跨天项目，需定期生成 Checkpoint。
*   **S (System)**：系统级变更，必须经过 Ops 审计。

### 3.3 自主等级 (Autonomy Ladder)
*   **L0**：只建议，不动手。
*   **L1**：可逆操作（写草稿、查资料），直接执行。
*   **L2**：有影响但可回滚（提 PR、改配置），执行后汇报。
*   **L3**：不可逆操作（发布、付款、发邮件），必须人类老板（用户）确认。
