# OPT 设计原则知识库

> 本文档由 KO（知识管理员）维护，记录 OPT 一人设计公司在实践中沉淀的核心设计原则。

## 1. Agent 设计原则

**物理隔离，各司其职**。每个 Agent 只做一件事，拥有独立的记忆（Workspace）、独立的性格（SOUL.md）和独立的技能（Skills）。这是防止"角色精神分裂"和上下文膨胀的根本方法。

**工具即能力边界**。Agent 能做什么完全由其工具集（`allowedTools`/`disallowedTools`）决定，没有任何后门。这是 Claude Code 的核心安全哲学。

**Fail-closed 优于 Fail-open**。所有安全相关的默认值都应该是最保守的。当不确定时，选择拒绝而非允许。

## 2. 记忆管理原则

**观点而非事实**。记忆是观点而非事实，不保存可通过工具实时推导的信息（如代码模式、Git 历史）。

**严格分离**。将记忆与计划/任务严格分离，避免混淆。

**去重优化**。自动合并重复的知识点，保持知识库的精简和高效。

## 3. 协作原则

**两步触发**。所有跨 Agent 的协作都必须在人类老板的视线范围内，通过可见锚点 + `sessions_send` 的两步触发机制完成。

**QAPS 任务结构**。所有跨 Agent 的任务传递，必须遵循 QAPS 结构（Question / Action / Context / Success Criteria）。

## 4. 产品设计原则

**简洁优于复杂**。界面和交互应尽可能简洁，每个元素都有其存在的理由。

**一致性优于创新**。在设计系统内保持一致性，比追求每次的视觉创新更重要。
